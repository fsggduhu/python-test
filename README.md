# 学生信息管理系统 (Student Management System)

### 🌟 项目简介
这是一个基于 Python 开发的轻量级命令行学生管理系统。系统采用模块化设计，支持教师与学生双视角切换，集成了数据持久化、自动化报表导出、正则表达式校验以及经典排序与搜索算法。

### 🚀 核心功能
* **权限分级**：
    * **教师端**：学生档案增删改查、成绩录入、全校科目均分/及格率分析、CSV 数据导出、系统日志审计。
    * **学生端**：个人档案查看、全校成绩排行榜查询、基于递归算法的“学习奖励积分”获取。
* **智能校验**：利用正则匹配（Regex）严格校验手机号、邮箱格式及年龄合法性。
* **高效搜索**：内置**快速排序 (Quick Sort)** 与 **二分查找 (Binary Search)**，确保在大数据量下依然保持高效性能。
* **数据持久化**：采用 JSON 格式存储本地数据，支持操作日志自动记录。

### 🛠️ 安装与运行步骤
1.  **依赖安装**：系统使用了 `prettytable` 库来美化终端输出。
    ```bash
    pip install prettytable
    ```
2.  **目录准备**：确保所有 `.py` 文件处于同一目录下。
3.  **启动系统**：
    ```bash
    python main.py
    ```
4.  **默认教师密码**：`123456`（可在 `config.py` 中修改）。

## 📐 设计说明

### 1. 软件架构
项目遵循**关注点分离 (SoC)** 原则，划分为五个核心模块：
* `models.py`：**模型层**。定义数据结构与核心业务对象。
* `storage.py`：**持久层**。处理 JSON 读写、CSV 导出与日志 IO。
* `utils.py`：**工具层**。封装校验逻辑与底层算法（排序、搜索、递归）。
* `config.py`：**配置层**。管理全局常量、路径及安全密钥。
* `main.py`：**控制层**。负责路由逻辑、菜单交互与权限验证。

### 2. 面向对象 (OOP) 设计
系统充分体现了 OOP 的三大特性：
* **继承**：`Student` 与 `Teacher` 均继承自基类 `Person`，复用基本属性。
* **封装**：学号 (`__student_id`) 等关键字段设为私有，通过 `@property` 暴露只读访问，防止意外篡改。
* **多态**：重写 `display_info()` 方法，根据对象类型（教师/学生）动态展示不同的身份信息。

### 3. 算法实现
为了优化查询与展示效率，系统实现了以下算法：
* **快速排序 (Quick Sort)**：采用分治策略，时间复杂度平均为 $O(n \log n)$，用于生成成绩排行榜。
* **二分查找 (Binary Search)**：针对已排序的学号列表进行检索，查找效率提升至 $O(\log n)$。
* **递归 (Recursion)**：在计算“斐波那契奖励积分”时展示了递归调用逻辑。
